# 1. Les facts

Les **Ansible facts** sont des données collectées automatiquement par **Ansible** sur vos systèmes distants, utilisées pour gérer ces systèmes de manière plus intelligente et adaptative. Ils sont collectés par le module **setup**.

Par défaut, **Ansible** exécute automatiquement ce module au début de chaque **play**, sauf si la collecte de **facts** est désactivée (avec `gather_facts: false`).

Une fois collectés, ces **facts** sont stockés dans la variable **ansible_facts** et peuvent être utilisés tout au long du **playbook** pour conditionner l'exécution des tâches, configurer les systèmes, etc.

Les **facts** incluent une variété d'informations système telles que les adresses **IP**, les systèmes de fichiers connectés, les détails du système d'exploitation, et plus encore.

Vous pouvez les afficher dans un **playbook** en utilisant `ansible.builtin.debug` :

```YAML
- name: Display facts
  ansible.builtin.debug:
    var: ansible_facts
```

Ou en utilisant la commande en ligne :

```SHELL
ansible <host> -m ansible.builtin.setup
```

## 1.1. Exemples

### Collecter des informations spécifiques avec les filtres

Dans les **playbooks** :

```YAML
- name: Collect memory and processor information
  ansible.builtin.setup:
    filter:
      - ansible_memory_mb
      - ansible_processor*
```

En ligne de commande :

```SHELL
ansible node1 -m ansible.builtin.setup -a "filter=ansible_distribution*" -i inventory.yml
```
